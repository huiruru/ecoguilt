Show the environmental impact of this coding session.

## Step 1: Find the session and get the numbers

The status line writes state to `/tmp/ecoguilt-{session_id}.json`. `CLAUDE_SESSION_ID` may not be available in skill context, so find the active state file by most recent modification time:

```bash
STATE_FILE=$(ls -t /tmp/ecoguilt-*.json 2>/dev/null | grep -v 'models\|test\|recommend' | head -1); echo "STATE: $STATE_FILE"; cat "$STATE_FILE" 2>/dev/null || echo '{}'
```

Note the session ID from the filename — you'll need it for the recommendation file. Extract it:
```bash
SESSION_ID=$(echo "$STATE_FILE" | sed 's|/tmp/ecoguilt-||; s|\.json||')
```

If the file is empty or missing, tell the user the status line hasn't recorded data yet.

The file contains: `input_tokens`, `output_tokens`, `total_tokens`, `total_kwh`, `co2_g`, `water_ml`, `model`, `cost_usd`, `input_kwh_per_token`, `output_kwh_per_token`.

These numbers are the single source of truth — they match what the status line displays. Do NOT recalculate energy, CO2, or water from tokens. Use the values from this file directly.

## Step 2: Get Not Diamond's recommendation

Read the cached recommendation using the same session ID from step 1:
```bash
cat "/tmp/ecoguilt-${SESSION_ID}-recommend.json" 2>/dev/null || echo '{}'
```

If the file is empty, missing, or stale, run the recommend script. First find the transcript:
```bash
ls -t ~/.claude/projects/*/${SESSION_ID}.jsonl 2>/dev/null | head -1
```

If that doesn't find a file, try:
```bash
ls -t ~/.claude/projects/*/*.jsonl 2>/dev/null | head -1
```

Then:
```bash
bash ${CLAUDE_SKILL_DIR}/../../scripts/recommend.sh "<transcript_path>" 2>/dev/null || echo '{"error": "failed"}'
```

The response is JSON with `model`, `provider`, and `messages_sent`, or `{"error": "no_api_key"}` if the user doesn't have `NOTDIAMOND_API_KEY` set.

## Step 3: Calculate the comparison

Read the model cache to get pricing and energy data for the recommended model:
```bash
cat /tmp/ecoguilt-models.json 2>/dev/null || echo '[]'
```

This file is auto-generated by `sync-models.sh` from the user's `models.json` + Not Diamond's API. Each entry has: `id`, `provider`, `display`, `input_price` ($/1M tokens), `output_price` ($/1M tokens), `input_kwh_per_token`, `output_kwh_per_token`, `available`.

The session state file has: `input_tokens`, `output_tokens`, `input_kwh_per_token`, `output_kwh_per_token`, `cost_usd`.

For the recommended model, look up its entry in the model cache by `id`, then calculate:
- cost = input_tokens × input_price/1M + output_tokens × output_price/1M
- kWh = input_tokens × rec_input_kwh + output_tokens × rec_output_kwh
- CO2 = kWh × 0.39 × 1000 (grams)
- water = kWh × 1800 (ml)
- miles driven (400g CO2 per mile)
- bottles of water (500ml each)

## Step 4: Present the bill

**Section 1 — what you used:**
State the raw numbers one per line: tokens, kWh, CO2, water, cost. Then the visual comparisons (miles driven, bottles of water, lightbulb hours, etc). Each line should be a scene the reader can picture.

Do NOT use abstract comparisons like grams of coal, ml of gasoline, or "a tree needs X days." Use: miles driven in a car going nowhere, bottles of water poured down a drain, lightbulbs left on in empty rooms, smartphone charges.

IMPORTANT: each comparison must describe a DIFFERENT resource. Do not repeat the same resource in two different ways (e.g. "2 bottles of water" and "half a day's drinking water" are the same thing — pick one). One line per resource: one for CO2, one for water, one for energy.

**Section 2 — what you could have used:**
If Not Diamond returned a recommendation different from the current model, show:
- "not diamond says: [display name] could have handled this."
- The same metrics recalculated for that model
- The delta: "that's Xg less CO2. X fewer bottles of water. $X cheaper."

If the recommended model IS the current model (or more expensive), say so: "not diamond agrees: this task needed [current model]."

If `no_api_key` error: show the impact numbers, then add:
"set NOTDIAMOND_API_KEY to see what a cheaper model would have saved. get a key at notdiamond.ai"

**Tone:** Deadpan. The numbers do the talking. No softening, no "great job for checking!" — just the bill and what could have been different.
